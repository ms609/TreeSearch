## What to Look For

### Bug Patterns (from past red-team rounds)

| Pattern | Where to check | Example |
|---------|---------------|---------|
| Missing `GetRNGstate()`/`PutRNGstate()` around `unif_rand()` | Any .cpp using randomness | ts_ratchet.cpp, ts_sector.cpp (both fixed) |
| `std::random_device{}()` ignoring `set.seed()` | Seeding of `std::mt19937` | ts_tbr.cpp, ts_drift.cpp (both fixed) |
| GCC-only builtins (`__builtin_popcountll`, `__builtin_ctzll`, `__builtin_huge_val`) | All .cpp/.h files | Fixed with `popcount64()`, `ctz64()`, `HUGE_VAL` wrappers |
| `.inc` file changes not triggering recompilation | `ts_fitch_na.inc`, `ts_fitch_na_incr.inc` | Must `touch src/ts_fitch.cpp` after `.inc` changes |
| Missing `TreeSearch:::` prefix in tests | `tests/testthat/test-ts-*.R` | Causes failures under `library()` vs `devtools::load_all()` |
| Arg count mismatch in `TreeSearch-init.c` | After adding/removing Rcpp params | Count must match `ts_rcpp.cpp` bridge signature |
| `R_PosInf` in Rcpp defaults | `R/RcppExports.R` after `compileAttributes()` | Use `-1.0` sentinel instead |
| Score regression after topology changes | Any search function | Compare C++ scores against MorphyLib reference |
| No revert on worsening move | Sectorial reinsertion, fuse exchange | Must save/restore topology on worse score |
| `active_mask`/`upweight_mask` not cleaned up | Ratchet perturbation restore paths | `PerturbSnapshot` should fully restore |

### Performance Patterns

| Pattern | Where to check |
|---------|---------------|
| Unbounded indirect scoring (missing `_bounded` variant) | Search inner loops in ts_tbr.cpp, ts_drift.cpp, ts_search.cpp |
| Full `score_tree()` where incremental would suffice | After clip/regraft in any search |
| `build_postorder()` called unnecessarily | After unclip or snapshot restore |
| Full-tree copy where save/restore suffices | Fuse exchange, sectorial reinsertion |
| Missing early termination in loops | Block iteration in scoring functions |

### Correctness Checks

1. **Score verification**: Pick 3-5 datasets, run `ts_driven_search` and
   compare best score against `TreeLength()` on the returned tree.
2. **Determinism**: Run the same search twice with `set.seed()`; results
   must match (serial mode only; parallel is non-deterministic by design).
3. **Edge cases**: 3-tip trees, single-character datasets, all-inapplicable
   characters, zero-weight characters after resampling.
4. **Memory safety**: Look for raw pointer arithmetic without bounds checks,
   `memcpy` with wrong sizes, vector indexing beyond `.size()`.

## Known Fragile Areas

1. **`ts_rcpp.cpp` + `TreeSearch-init.c`**: Shared by all agents. Append-only.
   Check arg counts match after any parameter change.
2. **`RcppExports.R` / `RcppExports.cpp`**: Regenerated by `compileAttributes()`.
   Must fix `concavity` defaults (replace `Inf` with `-1.0` sentinel) afterward.
3. **`.inc` files**: Build system doesn't track them. Changes require
   `touch src/ts_fitch.cpp`.
4. **Parallel code paths**: `ts_rng.h` thread_local pointers must be set before
   any search function calls `ts::make_rng()`. Check worker thread setup in
   `ts_parallel.cpp`.
5. **`init_from_edge` left-right convention**: First child â†’ left (fixed in
   integration review). If any code constructs edge matrices manually, verify
   this convention.
